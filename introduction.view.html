Dependencies

You can account for nested dependencies by creating a static function "dependencies" on the PHP-controller. No need to import your dependencies or taking care of nested dependencies. There are two scenarios you want to account for:

Reusability

Created a new component and used another component? Add the dependency to the composer.json of the component (using neoan3 publish). This way, whenever that component is used in another project, dependencies are installed along with it.

Inheritance

What makes vast-n3 so fast is the neoan3-core. In order to only load needed components, vast-n3's functions "vueComponent" & "vueComponents" look for a function "dependencies" which should return an array of component names. As an example you might want to look into the register component which requires "animation" and "modal". This means that whenever a neoan3 route uses "register", the components "animation" and "modal" are automatically imported.

Best practices

vast-n3 uses neoan3's methodology for development, collaboration and versioning. As such, make use of neoan3's publish logic and treat every component as it's own git repository. vast-n3 comes to you as a complete installer. But just like every other neoan3 project, you can add or update single components.
neoan3 add component login https://vast-n3/component-login.git
will install vast-n3's login component including its dependencies into any neoan3-project. However, be aware that we have opted not to use neoan3's singleton approach. This means that our components do require the vast-n3 frame as a hard-coded necessity.

The recommended way to take complete control over your project is to change the upstream of the component you are working on and pointing it to a fork you made.
